## 📝 세부계획

<details><summary>1차</summary>
- 3명/3명 팀분리

- 공지된 범위 개인공부 + 면접질문 3개 정도 준비

- 스터디 시간 맞춰서 디스코드 접속 - 각팀 방에서 모의면접 진행  
  - 모의면접은 특이사항 없으면 카메라 켜고 진행
  - 돌아가면서 질문자 역할 -> 질문에 대해 답변자 2명이 각각 순서대로 답변
  - 답변 내용(지식, 정보)에 대한 피드백 + 스피치, 태도, 등에 대한 피드백
  - 질문자 역할 변경 (3회 반복)

- 주차별 당번이 스터디 내용 MD 파일 정리
  - 개인적으로 공부한 내용
  - 스터디 진행하며 추가로 알게된 내용
  - 각 팀원 질문 + 답변 요약

- 정리한 MD문서는 스터디 다음날 오후 6시 전까지 각팀 디렉토리에 PR

- 각팀 당번 팀교환, 다음주 당번 및 스터디 범위 전달

</details>

#### 2차
- 3명/2명 팀분리 + 1차 인원 1명
- 9시팀 인원 1명이 10시팀 연속해서 진행
- <U>**10시에 5명 전체 모여서 다음주 주제 선정**</U>
  - 기존 스터디와 비슷한 유형의 CS문제
  - 프레임워크 + 언어 관련 문제(하단 질문리스트 중 5개 선택)
  - 기술적인 부분 외의 면접에서 나올만한 질문(예: 프로젝트 진행시 어려웠던점, 집중한부분 등)
- 질문은 각 주제별로 1~2개(질문자 한명당 3~5개의 질문 진행)
- 넓은 범위에 대한 공부보다는 특정 질문에 대한 <span style="color:orange"> **답변 플랜** </span> 준비

<br>

- 각팀 스터디 시간 맞춰서 디스코드 접속
  - 모의면접은 특이사항 없으면 카메라 켜고 진행
  - 돌아가면서 질문자 역할, 질문에 대해 답변자 2명이 각각 순서대로 답변
  - 질문자 차례가 끝나면 답변자 각각에 피드백 전달
  - 질문자 역할 변경 (3회 반복)

- 하루에 1팀, 2팀 2회 진행한 인원은 2팀 잔류
- 2팀 인원 1명은 1팀으로 이동

# 컨벤션

## 📌 MD 컨벤션
1. 예상 질문 토글로 MD에 같이 넣어주기
2. 최상단 README -> 스터디 주제에 자신의 MD url 연결해주기
3. 파일명은 (주제)_team1.md 형식

## 📌 Git 컨벤션
1. 한 가지 md에 대해 PR 진행하기

### 브랜치 명
- (브랜치 용도)/(주제)  
- 예외로 feature는 사용빈도가 높고 길이가 길어서 feat으로 단축하여 사용  
- 정리본 추가시 docs 대신 feat사용  
- develop브랜치가 따로 없으므로 main브랜치로 부터 신규 브랜치 생성  
- 각 주제별로 브랜치 생성 -> 한주에 여러개의 브랜치가 생성될수 있음.

예시)  
- feature/http  
- docs/readme

<br/>

### 커밋 메세지
- (동사 기본형): (내용)
- 난 첫 글자는 대문자로 주로 썼었는데 합의 필요
- 정리본 추가시엔 Feat사용
- 내용 파트는 영어/한글 자유 사용

예시)
- Feat: ip-address
- Fix: tcp/ip and source reference address
- Add: git ignore
- Chore: some typo and others
- Docs: 리드미 링크 수정
